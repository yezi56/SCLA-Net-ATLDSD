新的判断：

```text
只做 CFR 可能仍然是单点 gate，解释力和涨点都不够。
PDF 强调的有效模板是:
Token Mixer + FFN
A+B / A+B+C 双分支
低分辨率全局建模
高频/频域小目标增强
自适应融合

所以后续计划升级为:
先保输入分辨率，再做组件引导高频细化，最后才做轻量 CNN-Mamba-Frequency 融合。
```

新的训练顺序：

```text
第1步: Boundary1-384
结构:
  Boundary1 原结构，input_shape 384x384。
目的:
  不加新模块，只验证小病斑是否被 256x256 压没。

第2步: Boundary1-repeat
结构:
  Boundary1 原结构，seed 42 / 2026。
目的:
  验证 72.86 是否稳定。

第3步: CHFR-256
结构:
  Boundary1 + Component-guided High-Frequency Refinement。
位置:
  LBSB 后、cls_conv 前。
借鉴:
  PDF 的 HS-FPN 高频小目标、FreqConvMamba 频域增强、alpha 自适应融合。

  直接解决 alternaria / gray_spot / brown_spot 小病斑边界和纹理。

第4步: CHFR-384

  Boundary1-384 或 CHFR-256 接近成功时才跑。
```

## 新训练顺序

### 第1步：Context1

```text
结构:
主线1 + LBSB + LGLC

模块位置:
ASPP 后，decoder concat 前。

目的:
验证“全局叶片上下文 + 局部病斑细节”是否能继续提升 Boundary1。

判断:
如果 mIoU > 72.86:
  LGLC 进入最终候选。
如果 mIoU 接近 72.86，但 alternaria / gray_spot / brown_spot 任意两类提升:
  LGLC 作为小病斑候选。
否则:
  LGLC 不保留。
```

### 第2步：Refine1

```text
结构:
主线1 + LBSB + CFR

模块位置:
LBSB 后、cls_conv 前。

实现:
用 lesion_aux_head / boundary_aux_head / center_aux_head 的概率图生成 feedback gate。
gate 只调制 decoder feature，不改变标签、不做两阶段推理。

目的:
把辅助头从“只算 loss”升级成“参与推理的反馈模块”。

判断:
如果 mIoU > 72.86:
  CFR 进入最终候选。
如果 severity MAE < 0.01169 且 mIoU 不低于 72.60:
  CFR 可作为严重度友好模块。
否则:
  CFR 不保留。
```

### 第3步：UHF

```text
结构:
当前最强候选 + UHF

模块位置:
decoder feature 上，用 softmax entropy 或 lesion probability 找不确定区域。

目的:
只在边界/不确定区域增强高频，不全图加复杂 attention。

触发条件:
只有 Context1 / Refine1 仍无法超过 Boundary1，才做 UHF。
```

### 第4步：Severity-Calib

```text
结构:
最终结构 + severity consistency loss

目的:
专门看 severity MAE / grade accuracy 是否改善。

定位:
应用指标增强，不写成主结构创新。
```

## 论文主线

```text

```
