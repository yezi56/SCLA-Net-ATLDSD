## PDF 借鉴点如何落地

这轮不是把 PlugNPlay 模块当购物清单乱接，而是按 `论文创新指南2026` 的几个设计模板筛：

```text
1. Token Mixer + FFN:
   不替换 DeepLabV3+ 主体，只在 ASPP / decoder 附近插入轻量 mixer/refinement。

2. A+B / A+B+C:
   保留 Boundary1 的 CNN 边界能力，再试局部大核、多尺度或注意力分支。

3. 高频/小目标:
   小病斑更像局部纹理和边界问题，所以优先试大核选择性局部上下文，而不是完整 Mamba 双主干。

4. 自适应融合:
   候选模块必须轻量、可插拔，后续如果进入正式设计，要被 lesion/boundary/center component cues 门控。
```

本轮从 `ai-dawang/PlugNPlay-Modules` 及仓库已有 `src/modules/plugins` 中优先筛选：

```text
LSK:
  大核选择性空间上下文，贴合 PDF 的局部/多尺度 Token Mixer 方向。

CPCA:
  channel prior + 3/5/7 局部分支，贴合局部多尺度和空间注意力方向。

EMA:
  grouped efficient multi-scale attention，轻量但小样本信号不足。

SCSA:
  spatial-channel synergistic attention，参数很少但小样本信号为负。
```

## 2026-06-09 CHFR v0 快速探针
